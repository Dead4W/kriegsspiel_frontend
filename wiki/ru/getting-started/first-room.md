# Первая комната

## Дополнительные настройки

### Игровая дата

- Стартовая дата и время партии.
- Формат: `YYYY-MM-DD HH:mm:ss` (пример: `1882-06-12 09:00:00`).

### Ресурс-пак

- **URL ресурс-пака** — ссылка на JSON с балансом и модификаторами.
- Можно оставить стандартный URL по умолчанию.

### Карта

- `Essex 1882` и `Saint-Petersburg 1828` — готовые пресеты.
- `Custom map` — своя карта.

Для `Custom map`:

- **URL Карты** — прямая ссылка на изображение карты.
- **Метров на пиксель** — масштаб карты.

### URL Карты высот (опционально)

- Это ссылка на **height map** (карту высот): отдельное изображение, где яркость пикселя соответствует высоте местности.
- Если у вас есть исходная `.tiff` карта высот, её можно подготовить по координатам через скрипт:
  [generate_height_map.py](https://github.com/Dead4W/kriegsspiel_backend/blob/main/helpers/generate_height_map.py).

## Параметры комнаты

### Названия сторон

- **Название красной команды** — имя стороны Red в интерфейсе.
- **Название синей команды** — имя стороны Blue в интерфейсе.

### Информация и отчёты

- **Генерал обновляет информацию в прямой видимости** — если генерал визуально видит юнит на карте, этот юнит актуализируется на досках игроков. Это касается и своих, и вражеских юнитов.
- **Автоматическое обновление статов** — когда генерал получает рапорт, на доске игрока обновляется информация юнита, который отправил рапорт (позиция, HP, патроны и другие актуальные статы).

### Модификаторы боя

- **Модификаторы времени суток** — характеристики юнитов зависят от времени.
- **Модификаторы погоды** — характеристики юнитов зависят от погоды.

### Карта и логистика

- **Персональные карты игроков** — у каждого игрока своя доска. Если в команде Red больше одного игрока, у них будут разные фишки, чат и прочий контекст. Игрок видит и обновляет только свою доску.
- **Патроны не бесконечные** (`BETA`) — у юнитов ограниченный боезапас.

## Как пригласить друзей

1. После входа в комнату откройте слева сверху вкладку **Umpire / Судья**.
2. Там есть кнопки копирования ссылок для `umpire`, `red` и `blue`.
3. Отправляйте каждому участнику только его ссылку.
4. Игроки Red/Blue не получают доступ к другим доскам; полный доступ есть только у Umpire (Судьи).
