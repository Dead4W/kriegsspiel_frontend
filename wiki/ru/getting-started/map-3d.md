# 3D карта (object map + object meta)

![Пример заполнения полей 3D карты](/assets/wiki/first-room-3d-map-example.png)

Поля `Object map URL` и `Object meta URL` нужны для 3D генерации окружения: лесов, дорог, рек, мостов и зданий.

## Зачем это нужно

- Без `object map` + `object meta` 3D карта будет беднее по окружению.
- Эти данные влияют на визуальную сцену и служебные слои (например, лес).
- Поля работают только в паре: если заполнено одно, второе обязательно.

## Object map URL

- Прямая ссылка на маску объектов (PNG/JPEG).
- Делайте маску в том же размере, что и `Map URL`, чтобы избежать смещений.
- Каждый объект рисуйте чистым RGB-цветом, без градиентов и антиалиасинга.

## Object meta URL

- Прямая ссылка на JSON с ключом `entity_to_color`.
- `entity_to_color` задает соответствие: `тип объекта -> RGB цвет`.

Минимальный пример:

```json
{
  "entity_to_color": {
    "forest": [234, 20, 255],
    "road": [20, 255, 38],
    "good_road": [107, 20, 255],
    "river": [255, 20, 71],
    "bridge": [255, 175, 20],
    "building": [20, 139, 255],
    "red_building": [208, 255, 20]
  }
}
```

## Быстрый пайплайн подготовки

1. Создайте отдельную маску объектов того же размера, что и основная карта.
2. Закрасьте области объектов цветами из вашего `entity_to_color`.
3. Сохраните маску как PNG/JPEG, мету как JSON.
4. Разместите файлы по прямым URL.
5. Вставьте оба URL в `Custom map` при создании комнаты.
