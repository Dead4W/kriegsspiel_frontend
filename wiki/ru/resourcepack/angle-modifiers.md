## `angleModifiers`

Путь: `angleModifiers[]`

Это модификаторы угла атаки. Они нужны, чтобы менять урон в зависимости от того, атакует юнит вверх, вниз или по ровной поверхности.

Например, атаку сверху вниз можно сделать сильнее, а атаку снизу вверх - слабее.

Угол рассчитывается по:

- высоте атакующего
- высоте цели
- горизонтальной дистанции

---

## ПАРАМЕТРЫ

Каждый элемент в `angleModifiers[]` поддерживает:

- `angle` (обязательно, number) - угол в градусах
- `modifier` (обязательно, number) - множитель урона для этого угла

Простыми словами:

- отрицательный `angle` - атака идет снизу вверх
- `angle = 0` - цель находится примерно на одном уровне
- положительный `angle` - атака идет сверху вниз

---

## ФОРМАТ

```json
{
  "angleModifiers": [
    { "angle": -6, "modifier": 0.5 },
    { "angle": -3, "modifier": 0.8 },
    { "angle": 0, "modifier": 1.0 },
    { "angle": 4, "modifier": 1.2 }
  ]
}
```

Каждая запись задает точку на шкале углов.

---

## КАК ЭТО РАБОТАЕТ

- если угол меньше минимального из таблицы, берется первое значение
- если угол больше максимального, берется последнее значение
- если угол находится между двумя точками, игра рассчитывает промежуточное значение

Если таблица пустая, используется множитель `1.0`.

---

## ПРИМЕР

```json
{
  "angleModifiers": [
    { "angle": -6, "modifier": 0.5 },
    { "angle": -3, "modifier": 0.8 },
    { "angle": 0, "modifier": 1.0 },
    { "angle": 4, "modifier": 1.2 }
  ]
}
```

---

## ОБЪЯСНЕНИЕ ПРИМЕРА

- `angle: -6` и `modifier: 0.5` означает, что при атаке снизу вверх под таким углом урон уменьшается до 50%.
- `angle: -3` и `modifier: 0.8` означает, что при менее крутой атаке снизу вверх урон уменьшается до 80%.
- `angle: 0` и `modifier: 1.0` означает обычный урон без изменений.
- `angle: 4` и `modifier: 1.2` означает, что при атаке сверху вниз урон увеличивается до 120%.
- Если фактический угол окажется, например, между `0` и `4`, игра возьмет промежуточное значение между `1.0` и `1.2`.
