## `environment`

Путь: `environment.states[]`

Это состояния окружения. Они нужны, чтобы описывать, как местность, укрытия или укрепления влияют на юнитов.

Например, лес может замедлять движение и снижать входящий урон, а дом может давать укрытие и помогать юниту лучше держать строй.

---

## ПАРАМЕТРЫ

Каждый элемент в `environment.states[]` поддерживает:

- `id` (обязательно, string) - уникальный ID состояния окружения, например `in_forest`
- `title` (опционально, string) - отображаемое имя состояния
- `icon` (опционально, string) - короткая иконка или подпись для интерфейса
- `isRoute` (опционально, boolean) - помечает состояние как связанное с дорогой
- `params` (опционально, object) - дополнительные параметры для отдельных игровых систем
- `multipliers` (опционально, object) - общие модификаторы характеристик
- `byTypes` (опционально, object) - отдельные модификаторы для конкретных типов юнитов

В `multipliers` и `byTypes` можно использовать стандартные характеристики:

- `damage`
- `takeDamageMod`
- `speed`
- `attackRange`
- `visionRange`

Все значения должны быть числовыми.

В `params` движок использует:

- `moraleCheckMod` - модификатор проверки морали от этого состояния окружения

Если на юнита действует несколько состояний окружения с `moraleCheckMod`, используется самое сильное значение по модулю.

---

## ФОРМАТ `multipliers`

`multipliers` - это общие модификаторы, которые действуют на все юниты в этом состоянии окружения.

```json
{
  "multipliers": {
    "takeDamageMod": 0.6,
    "speed": 0.8
  }
}
```

Например:

- `takeDamageMod: 0.6` уменьшает входящий урон до 60%
- `speed: 0.8` уменьшает скорость до 80%

---

## ФОРМАТ `byTypes`

`byTypes` позволяет задать отдельные значения для конкретных типов юнитов.

```json
{
  "byTypes": {
    "artillery": {
      "speed": 0.5
    }
  }
}
```

Это значит, что для артиллерии в этом окружении будет использоваться своя скорость.

Если одна и та же характеристика указана и в `multipliers`, и в `byTypes`, то для этого типа юнита используется значение из `byTypes`.

---

## ПРИМЕР

```json
{
  "environment": {
    "states": [
      {
        "id": "in_forest",
        "title": "Forest",
        "icon": "F",
        "isRoute": true,
        "multipliers": {
          "takeDamageMod": 0.6,
          "speed": 0.8
        },
        "byTypes": {
          "artillery": {
            "speed": 0.5
          }
        }
      },
      {
        "id": "in_house",
        "title": "House",
        "icon": "H",
        "params": {
          "moraleCheckMod": 2
        },
        "multipliers": {
          "takeDamageMod": 0.4
        }
      }
    ]
  }
}
```

---

## ОБЪЯСНЕНИЕ ПРИМЕРА

- `in_forest` описывает состояние юнита в лесу.
- `takeDamageMod: 0.6` означает, что в лесу юнит получает меньше урона.
- `speed: 0.8` означает, что в лесу юнит двигается медленнее.
- Для `artillery` дополнительно задано `speed: 0.5`, поэтому артиллерия в лесу движется еще медленнее, чем остальные.
- `in_house` описывает состояние юнита в доме или укрытии.
- `takeDamageMod: 0.4` означает, что в доме юнит защищен еще лучше и получает только 40% входящего урона.
- `moraleCheckMod: 2` означает, что укрытие помогает юниту лучше проходить проверку морали.
