## `formations`

Путь: `formations.types[]`

Это строи юнитов. Они нужны, чтобы задавать разные режимы построения и их влияние на характеристики.

Например, один строй может давать больше скорости, а другой - повышать эффективность атаки.

---

## ПАРАМЕТРЫ

Каждый элемент в `formations.types[]` поддерживает:

- `id` (обязательно, string) - уникальный ID строя
- `title` (опционально, string) - отображаемое имя строя
- `icon` (опционально, string) - короткая иконка или подпись для интерфейса
- `multipliers` (опционально, object) - модификаторы характеристик для этого строя

В `multipliers` можно использовать стандартные характеристики:

- `damage`
- `takeDamageMod`
- `speed`
- `attackRange`
- `visionRange`

Все значения должны быть числовыми.

---

## ФОРМАТ `multipliers`

`multipliers` - это модификаторы характеристик, которые действуют, пока юнит находится в этом строю.

```json
{
  "multipliers": {
    "speed": 1.2,
    "damage": 0.9
  }
}
```

Например:

- `speed: 1.2` увеличивает скорость до 120%
- `damage: 0.9` уменьшает урон до 90%

---

## ПРИМЕР

```json
{
  "formations": {
    "types": [
      {
        "id": "line",
        "title": "Line",
        "icon": "L",
        "multipliers": {
          "damage": 1.1
        }
      },
      {
        "id": "column",
        "title": "Column",
        "icon": "C",
        "multipliers": {
          "speed": 1.25,
          "takeDamageMod": 1.2
        }
      }
    ]
  }
}
```

---

## ОБЪЯСНЕНИЕ ПРИМЕРА

- `line` - строй, который повышает урон.
- `damage: 1.1` означает, что юнит в этом строю наносит 110% от базового урона.
- `column` - строй, который помогает быстрее двигаться.
- `speed: 1.25` означает, что юнит в колонне движется на 25% быстрее.
- `takeDamageMod: 1.2` означает, что юнит в колонне получает 120% входящего урона, то есть становится более уязвимым.
