## `moraleCheck`

Путь: `moraleCheck`

Этот раздел настраивает проверку морали после получения урона.

Проще говоря: когда юнит получает урон, игра проверяет, сохраняет ли он строй и боеспособность, или начинает отступать, бежать или распадается.

Результат проверки может быть таким:

- `pass` - юнит проходит проверку морали и продолжает действовать как обычно
- `retreat` - юнит не выдерживает давление и начинает отступать
- `flee` - юнит обращается в бегство и теряет боеспособность сильнее, чем при обычном отступлении
- `disband` - юнит полностью распадается и выбывает из боя

---

## ПАРАМЕТРЫ

Раздел `moraleCheck` поддерживает:

- `dice` - настройки броска кубиков
- `commander` - штраф, если рядом нет дружественного командира
- `lossPenalties` - дополнительные штрафы за потери
- `outcomes` - возможные результаты проверки морали
- `effects` - что именно происходит после результата

Внутри этих полей используются такие основные параметры:

- `dice.count` - сколько кубиков бросается
- `dice.sides` - сколько граней у кубика
- `commander.radiusMeters` - радиус проверки наличия командира
- `commander.penalty` - штраф, если командира рядом нет
- `lossPenalties[].lossesMoreThan` - порог потерь
- `lossPenalties[].penalty` - штраф за превышение этого порога
- `lossPenalties[].key` - ID или подпись этого штрафа
- `outcomes[].minTotal` - минимальное итоговое значение для результата
- `outcomes[].id` - ID результата (`pass`, `retreat`, `flee`, `disband`)
- `effects.retreatDurationSeconds` - длительность отступления
- `effects.fleeDurationMultiplier` - множитель длительности бегства
- `effects.fleeHpMultiplier` - множитель HP при бегстве

---

## ФОРМАТ

```json
{
  "moraleCheck": {
    "dice": { "count": 2, "sides": 6 },
    "commander": { "radiusMeters": 300, "penalty": -2 },
    "lossPenalties": [
      { "lossesMoreThan": 0.25, "penalty": -2, "key": "losses>25%" }
    ],
    "outcomes": [
      { "minTotal": 0, "id": "pass" },
      { "minTotal": -2, "id": "retreat" },
      { "minTotal": -5, "id": "flee" },
      { "minTotal": -9999, "id": "disband" }
    ],
    "effects": {
      "retreatDurationSeconds": 43200,
      "fleeDurationMultiplier": 2,
      "fleeHpMultiplier": 0.5
    }
  }
}
```

---

## КАК ЭТО РАБОТАЕТ

Во время проверки морали игра:

1. Бросает кубики.
2. Учитывает мораль самого юнита.
3. Проверяет, есть ли рядом дружественный командир.
4. Учитывает, сколько потерь уже понес юнит.
5. Добавляет другие модификаторы, если они есть, например от окружения или типа юнита.
6. Сравнивает итоговое значение со списком результатов в `outcomes`.

Чем хуже итоговое значение, тем тяжелее результат проверки.

---

## ПРИМЕР

```json
{
  "moraleCheck": {
    "dice": { "count": 2, "sides": 6 },
    "commander": { "radiusMeters": 300, "penalty": -2 },
    "lossPenalties": [
      { "lossesMoreThan": 0.25, "penalty": -2, "key": "losses>25%" }
    ],
    "outcomes": [
      { "minTotal": 0, "id": "pass" },
      { "minTotal": -2, "id": "retreat" },
      { "minTotal": -5, "id": "flee" },
      { "minTotal": -9999, "id": "disband" }
    ],
    "effects": {
      "retreatDurationSeconds": 43200,
      "fleeDurationMultiplier": 2,
      "fleeHpMultiplier": 0.5
    }
  }
}
```

---

## ОБЪЯСНЕНИЕ ПРИМЕРА

- Игра бросает `2` шестигранных кубика.
- Если рядом нет дружественного командира в радиусе `300` метров, юнит получает штраф `-2`.
- Если потери юнита больше `25%`, он получает еще штраф `-2`.
- Если итог проверки получился `0` или выше, юнит сохраняет строй (`pass`).
- Если итог получился ниже, могут сработать более тяжелые результаты: `retreat`, `flee` или `disband`.
- `retreat` заставляет юнит отступать на `43200` секунд.
- `flee` делает бегство более долгим и уменьшает HP юнита до `50%` от текущего значения по правилу `fleeHpMultiplier`.
- `disband` означает, что юнит полностью выбывает из боя.

## ПРИМЕР СИМУЛЯЦИИ

Представим юнит пехоты:

- у него было `100` HP
- после атаки он получил сильный урон и у него осталось `60` HP
- это значит, что юнит потерял `40%` здоровья
- рядом с ним нет дружественного командира

Что происходит дальше:

1. Игра запускает проверку морали после получения урона.
2. Бросаются `2d6`.
3. Так как потери больше `25%`, применяется штраф из `lossPenalties`.
4. Так как рядом нет командира, применяется еще один штраф.
5. В этом примере юнит получает два штрафа по `-2`, значит общий штраф равен `-4`.
6. Итог проверки считается как: результат `2d6 - 4`.

Тогда:

- для `pass` нужен итог `0` или выше, значит на `2d6` нужно выбросить `4` или больше
- для `retreat` нужен итог от `-2` до `-1`, значит на `2d6` нужно выбросить `2` или `3`
- `flee` в этом примере получить нельзя, потому что даже минимальный бросок `2` даст итог `-2`
- `disband` в этом примере тоже получить нельзя

Это значит, что в таком примере после тяжелого урона и без поддержки командира юнит либо сохранит строй, либо начнет отступать.
