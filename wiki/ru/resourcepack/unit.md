## `units`

Путь: `units.types[]`

Это типы юнитов. В этом разделе задаются все юниты, которые можно использовать в игре.

Например, здесь можно описать пехоту, артиллерию, генерала или другие типы войск.

---

## ПАРАМЕТРЫ

Каждая запись в `units.types[]` поддерживает:

- `id` (обязательно, string) - уникальный ID типа юнита (`"infantry"`, `"artillery"` и т.д.)
- `title` (опционально, string) - отображаемое имя юнита
- `stats` (обязательно, object) - базовые числовые характеристики
- `abilities` (опционально, string[]) - ID способностей, доступных юниту
- `defaultFormation` (опционально, string) - ID строя из `formations.types[]`
- `params` (опционально, object) - дополнительные параметры поведения и отображения

---

## ОБЯЗАТЕЛЬНЫЕ ХАРАКТЕРИСТИКИ

Все поля ниже обязательны и должны быть числовыми:

- `maxHp`
- `damage`
- `speed`
- `takeDamageMod`
- `attackRange`
- `visionRange`
- `ammoMax`

Если хотя бы одна обязательная характеристика отсутствует или указана неправильно, такой юнит не сможет нормально использоваться.

---

## ПАРАМЕТРЫ `params`

В `params` можно задавать дополнительные настройки.

Для отображения:

- `textureUrl` (string) - путь к спрайту/текстуре
- `renderIcon` (string) - запасной символ, если текстура недоступна
- `renderWidthMult` (number, по умолчанию `1`) - множитель ширины изображения юнита
- `renderHeightMult` (number, по умолчанию `1`) - множитель высоты изображения юнита

Для боя и выбора целей:

- `priorityTargets` (number) - команда атаки выбирает только ближайшие N целей
- `attackIgnoreHeightModifier` (boolean) - отключает модификатор урона по углу высоты для этого атакующего
- `attackIgnoreTargetEnvs` (string[]) - ID окружений цели, которые запускают специальный множитель
- `attackIgnoreTargetEnvMult` (number, по умолчанию `2`) - применяется при совпадении окружения цели

Для морали:

- `moraleCheckMod` (number) - модификатор типа юнита, используемый в проверках морали

Если `textureUrl` указан как относительный путь, он считается относительно загруженного ResourcePack.

---

## ФОРМАТ `stats`

`stats` - это базовые характеристики юнита.

```json
{
  "stats": {
    "maxHp": 64,
    "damage": 1,
    "speed": 80,
    "takeDamageMod": 1,
    "attackRange": 2000,
    "visionRange": 1000,
    "ammoMax": 10
  }
}
```

Эти значения задают основу юнита, а уже потом на них могут влиять строи, окружение, погода и способности.

---

## ДОПОЛНИТЕЛЬНО

UI спавна следует порядку в `units.types[]`, затем гарантирует наличие безопасных значений по умолчанию:

- всегда включает `general`
- внутренне гарантирует `messenger`, но он не показывается в обычном порядке списка спавна

## ПРИМЕР

```json
{
  "units": {
    "types": [
      {
        "id": "infantry",
        "title": "Infantry",
        "stats": {
          "maxHp": 64,
          "damage": 1,
          "speed": 80,
          "takeDamageMod": 1,
          "attackRange": 2000,
          "visionRange": 1000,
          "ammoMax": 10
        },
        "abilities": ["double_time_move"],
        "defaultFormation": "line",
        "params": {
          "priorityTargets": 3,
          "renderIcon": "I",
          "textureUrl": "units/infantry.png"
        }
      }
    ]
  }
}
```

---

## ОБЪЯСНЕНИЕ ПРИМЕРА

- `infantry` - это тип юнита "пехота".
- В `stats` задаются его базовые характеристики: здоровье, урон, скорость, дальность атаки, обзор и запас боеприпасов.
- `abilities: ["double_time_move"]` означает, что этот юнит может использовать указанную способность.
- `defaultFormation: "line"` означает, что по умолчанию юнит появляется или действует в строю `line`.
- `priorityTargets: 3` означает, что при атаке юнит будет рассматривать ближайшие цели в рамках этого лимита.
- `renderIcon: "I"` задает простой символ для отображения.
- `textureUrl: "units/infantry.png"` задает изображение юнита.
