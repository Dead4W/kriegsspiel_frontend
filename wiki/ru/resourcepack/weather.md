## `weather`

Путь: `weather.conditions[]`

Это погодные условия. Они нужны для глобальных модификаторов характеристик, например чтобы уменьшить обзор в туман или снизить дальность атаки в плохую погоду.

Также погода может включать визуальный эффект, чтобы игрок видел текущее состояние карты.

---

## ПАРАМЕТРЫ

Каждый элемент в `weather.conditions[]` поддерживает:

- `id` (обязательно, string) - уникальный ID погоды (`clear`, `fog` и т.д.)
- `title` (опционально, string) - отображаемое имя погоды
- `multipliers` (опционально, object) - модификаторы характеристик, пока активна эта погода
- `effect` (опционально, object) - визуальный погодный эффект

Параметр `effect` поддерживает такие варианты:

- `{"type": "fog", "mult": 1.0}` - эффект тумана
- `{"type": "clouds"}` - эффект облаков

---

## ФОРМАТ `multipliers`

`multipliers` - это объект с числовыми модификаторами характеристик.

```json
{
  "multipliers": {
    "visionRange": 0.7,
    "attackRange": 0.85
  }
}
```

Можно использовать стандартные параметры модификаторов, например:
`damage`, `takeDamageMod`, `speed`, `attackRange`, `visionRange`.

Все значения должны быть числовыми.

---

## ПРИМЕР

```json
{
  "weather": {
    "conditions": [
      {
        "id": "clear",
        "title": "Clear",
        "multipliers": {}
      },
      {
        "id": "fog",
        "title": "Fog",
        "multipliers": {
          "visionRange": 0.6,
          "attackRange": 0.8
        },
        "effect": {
          "type": "fog",
          "mult": 1.1
        }
      }
    ]
  }
}
```

---

## ОБЪЯСНЕНИЕ ПРИМЕРА

- `clear` - обычная ясная погода без модификаторов.
- `fog` - туман, который меняет характеристики юнитов.
- `visionRange: 0.6` уменьшает обзор до 60% от базового.
- `attackRange: 0.8` уменьшает дальность атаки до 80% от базовой.
- `effect.type: "fog"` включает визуальный эффект тумана.
- `effect.mult: 1.1` задает интенсивность этого эффекта.
